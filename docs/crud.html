1. Visão Geral

Este documento descreve a implementação das operações de criação (Create) e atualização (Update) de transações financeiras no sistema Best Financeiro, atendendo aos requisitos funcionais RF02 e RF03.

RF02: O sistema deve permitir o registro de receitas e despesas com valor, data, descrição e categoria.
RF03: O sistema deve permitir a edição de transações existentes.

3. Banco de Dados

Tabela: transacoes

Coluna Tipo Restrição Descrição
id INTEGER PRIMARY KEY AUTOINCREMENT Identificador único da transação
usuario_id INTEGER NOT NULL, FOREIGN KEY Referência ao usuário dono da transação
tipo TEXT NOT NULL "receita" ou "despesa"
valor REAL NOT NULL Valor monetário em R$
data TEXT NOT NULL Data no formato DD/MM/YYYY
descricao TEXT NOT NULL Descrição da transação
categoria TEXT NOT NULL Categoria (ex: moradia, trabalho, alimentação)

SQL de criação

CREATE TABLE IF NOT EXISTS transacoes ( id INTEGER PRIMARY KEY AUTOINCREMENT, usuario_id INTEGER NOT NULL, tipo TEXT NOT NULL, valor REAL NOT NULL, data TEXT NOT NULL, descricao TEXT NOT NULL, categoria TEXT NOT NULL, FOREIGN KEY (usuario_id) REFERENCES usuarios(id) );

5. DAO — CrudDAO.java

Camada de acesso a dados. Utiliza PreparedStatement para prevenir SQL Injection.

5.1 Método inserir() — CREATE

public boolean inserir(Crud transacao)
  • Parâmetro: objeto Crud com os dados da transação
  • Retorno: true se inseriu com sucesso, false em caso de erro
  • SQL: INSERT INTO transacoes (usuario_id, tipo, valor, data, descricao, categoria) VALUES (?, ?, ?, ?, ?, ?)
Utiliza PreparedStatement com parâmetros ? para prevenir injeção SQL.

5.2 Método atualizar() — UPDATE

public boolean atualizar(Crud transacao)
  • Parâmetro: objeto Crud com o id da transação e os novos dados
  • Retorno: true se atualizou com sucesso, false em caso de erro
  • SQL: UPDATE transacoes SET tipo = ?, valor = ?, data = ?, descricao = ?, categoria = ? WHERE id = ?
O WHERE id = ? garante que apenas a transação específica seja alterada.

6.1 Registrar Transação (Create)

Login: Usuário faz login e o sistema obtém seu userId
2
Menu Principal: Seleciona opção "1 - Registrar transação"
3
Entrada de dados: Informa tipo, valor, data, descrição e categoria

6.2 Editar Transação (Update)

1
Menu Principal: Seleciona opção "2 - Editar transação"
2
Identificação: Informa o ID da transação a editar
3
Novos dados: Informa o novo tipo, valor, data, descrição e categoria
4
Persistência: Sistema cria objeto Crud, define o ID e chama CrudDAO.atualizar()
5
Feedback: Exibe "Transacao atualizada com sucesso!" ou mensagem de erro

8. Alterações Realizadas

UsuarioDAO.login()

O método login() foi alterado para retornar int (ID do usuário) em vez de boolean, permitindo que o sistema vincule as transações ao usuário logado.

Antes Depois
public boolean login(...)
Retornava true / false		 public int login(...)
Retorna o id do usuário ou -1

Tratamento de valor numérico

A leitura do campo valor usa Double.parseDouble() com conversão de vírgula para ponto, garantindo compatibilidade com o locale brasileiro (PT-BR).

double valor = Double.parseDouble( scanner.nextLine().trim().replace(",", ".") );

10. Segurança

  • SQL Injection: Prevenido com PreparedStatement e parâmetros ?
  • Validação de entrada: Valor numérico tratado com try/catch para NumberFormatException
  • Integridade referencial: FOREIGN KEY (usuario_id) garante que transações pertencem a usuários válidos
  • Conexão segura: Uso de try-with-resources garante fechamento automático das conexões
