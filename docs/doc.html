1. Documento de Visão e Escopo Visão geral

1.1 Definição do objetivo do sistema

O sistema Gestão Financeira tem como objetivo oferecer ao usuário um controle completo sobre suas finanças pessoais, permitindo o registro de receitas e despesas, a visualização detalhada dos gastos por categoria e período, e a projeção de gastos futuros com base no histórico. O sistema busca transformar dados financeiros em informações úteis para que o usuário tome decisões mais conscientes sobre seu dinheiro.

1.2 Público-alvo

Pessoas físicas que desejam organizar e acompanhar suas finanças pessoais, entender seus padrões de consumo e identificar oportunidades de economia no dia a dia.

1.3 Problema que o sistema resolve

Muitas pessoas não têm clareza sobre para aonde seu dinheiro está indo, gastam mais do que deveriam e chegam ao fim do mês sem entender por que o saldo está negativo. O sistema resolve esse problema ao centralizar todas as movimentações financeiras em um só lugar, categorizando os gastos e mostrando comparativos entre meses — ajudando o usuário a enxergar onde pode economizar e quanto pode perder no mês seguinte se mantiver os hábitos.

1.4 Limitações do sistema

  • Funciona exclusivamente via terminal, sem interface gráfica web ou mobile.
  • Não oferece suporte a múltiplas moedas.
  • Não envia notificações ou alertas automáticos fora do sistema.
  • As projeções de gastos futuros são estimativas simples baseadas no histórico, sem uso de inteligência artificial.

2. Levantamento de Requisitos Especificação

2.1 Requisitos Funcionais

  • RF01: O sistema tem que ter cadastro e login de usuários.
  • RF02: O sistema deve permitir o registro de receitas e despesas com valor, data, descrição e categoria.
  • RF03: O sistema deve permitir edição e exclusão das transações feitas no sistema.
  • RF04: O sistema deve listar todas as movimentações financeiras feitas no sistema pelo usuário.
  • RF05: O sistema deve categorizar as transações.
  • RF06: O sistema deve exibir o saldo atual do usuário com base nas receitas e despesas.
  • RF07: O sistema deve apresentar opções de exportação CSV ao usuário.
  • RF08: O sistema deve apresentar relatórios de gastos seja ele: por categoria ou total.
  • RF09: O sistema deve apresentar uma projeção de gastos com base no que ele gastou durante o mês.
  • RF10: O sistema deve sugerir onde o usuário pode economizar com base nos padrões de consumo.

2.2 Requisitos Não Funcionais

  • RNF01: O sistema deve ser desenvolvido em Java.
  • RNF02: O sistema deve utilizar banco de dados relacional.
  • RNF03: O sistema deve ter documentação de todas as partes.
  • RNF04: As senhas dos usuários devem ser armazenadas com criptografia hash.
  • RNF05: O sistema deve operar apenas em Real.
  • RNF06: O sistema deve responder às operações do usuário em tempo hábil.
  • RNF07: O sistema deve funcionar apenas via terminal.
  • RNF08: Deve ser possível adicionar novos usuários sem impacto significativo de desempenho.
